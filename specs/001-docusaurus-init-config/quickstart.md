# Quickstart: Docusaurus Project Setup and Content Creation

This guide provides a quick overview of how to set up and start working with the Docusaurus project for the AI-Native Textbook. It covers initialization (if necessary), starting the development server, and adding new content.

## 1. Project Initialization (if Docusaurus is not yet present)

If Docusaurus is not already initialized in the `/frontend` directory, you can set it up using the following steps. **Note: The project is designed to automatically handle initialization or validation, so this step is primarily for manual intervention if needed.**

1.  **Navigate to the frontend directory**:
    ```bash
    cd frontend
    ```
2.  **Initialize Docusaurus**:
    Use the Docusaurus CLI to create a new project. Choose the `classic` template.
    ```bash
    npx create-docusaurus@latest . classic --typescript
    ```
    This command initializes Docusaurus in the current directory (`.`), ensuring all necessary files like `docusaurus.config.ts`, `sidebars.ts`, `package.json`, etc., are created.

## 2. Install Dependencies

Once the project is initialized (or if it already exists), install the required Node.js dependencies:

```bash
cd frontend
npm install
# or yarn install, if you prefer yarn
```

## 3. Start the Development Server

To see your Docusaurus site in action, start the local development server:

```bash
cd frontend
npm start
# or yarn start
```

This will open your browser to `http://localhost:3000` (or another available port) where you can view your Docusaurus site. The server features hot-reloading, so changes to your content or configuration will automatically refresh the browser.

## 4. Understanding the Core Configuration Files

-   **`docusaurus.config.ts`**: The main configuration file. Customize site metadata, navbar, theme settings, and plugin configurations here. Refer to `data-model.md` for a detailed structure.
-   **`sidebars.ts`**: Defines the structure of your documentation sidebar. This is where you configure modules and chapters. Refer to `data-model.md` for a detailed structure.
-   **`src/css/custom.css`**: For custom styling. Use this file to override Docusaurus's default styles or add new styles to ensure clean typography, proper spacing, and mobile responsiveness.

## 5. Adding New Content (Modules and Chapters)

The textbook content resides in the `docs/` directory within `/frontend`.

### To Add a New Module:

1.  **Create a new directory** inside `frontend/docs/` for your module (e.g., `frontend/docs/New-Module-Name`).
2.  **Add `_category_.json`**: Inside your new module directory, create a `_category_.json` file to define its display name and position in the sidebar.
    Example `frontend/docs/New-Module-Name/_category_.json`:
    ```json
    {
      "label": "New Module Name",
      "position": 5,
      "link": {
        "type": "generated-index",
        "description": "Learn about the new module."
      }
    }
    ```
3.  **Update `sidebars.ts` (if using explicit sidebar configuration)**: If you're not using autogenerated sidebars, you might need to add your new module as a category in `frontend/sidebars.ts`. However, for a content-heavy textbook, autogenerated sidebars are often preferred.

### To Add a New Chapter:

1.  **Create a new Markdown file** (`.md`) inside the relevant module directory (e.g., `frontend/docs/Module-1-ROS 2/new-chapter-title.md`).
2.  **Add Front Matter**: At the top of your Markdown file, include front matter for metadata.
    Example `frontend/docs/Module-1-ROS 2/new-chapter-title.md`:
    ```markdown
    ---
    id: new-chapter-title
    title: New Chapter Title
    sidebar_label: New Chapter Title
    ---

    This is the content of your new chapter.
    ```
    -   `id`: Unique identifier for the document. Defaults to the filename if not specified.
    -   `title`: The title displayed on the page.
    -   `sidebar_label`: The label shown in the sidebar.

3.  **Embed Static Assets**:
    -   For images in `frontend/static/img/`: `![Alt Text](/img/your-image.png)`
    -   For images within the same module directory: `![Alt Text](./your-image.png)`

## 6. Project Validation

To confirm the Docusaurus project is set up correctly according to the `spec.md`:

1.  **Run `npm start` in `/frontend`**.
2.  **Verify Homepage**: Ensure the Docusaurus homepage renders without errors.
3.  **Check Navigation**:
    -   Confirm the navbar displays correctly.
    -   Verify the sidebar contains 4 modules, each with its respective chapters, totaling 14 chapters.
    -   Navigate through all modules and chapters to ensure all links work.
4.  **Mobile Responsiveness**: Test the site on a mobile browser or by resizing your desktop browser window to ensure layout and readability are maintained.
5.  **Static Assets**: If content with images/diagrams is present, ensure they load and display correctly.

This quickstart should enable you to begin contributing content and verifying the Docusaurus setup efficiently.
