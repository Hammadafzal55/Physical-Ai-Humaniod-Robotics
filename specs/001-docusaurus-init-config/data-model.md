# Data Model: Docusaurus Configuration Entities

This document describes the structure of key Docusaurus configuration files relevant to the "Docusaurus Initialization & Configuration" specification. These files define how the textbook content is organized, navigated, and visually presented.

## 1. `docusaurus.config.ts`

This is the main configuration file for the Docusaurus site. It's a TypeScript file that exports a default configuration object.

### Key Sections and Their Structure:

-   **`title` (string)**: The title of your website.
-   **`tagline` (string)**: A short description/tagline for your website.
-   **`url` (string)**: The URL of your website.
-   **`baseUrl` (string)**: The base URL for your project. Must have a trailing slash.
-   **`onBrokenLinks` (string)**: Action to take when a broken link is found (e.g., `'throw'`, `'warn'`, `'ignore'`).
-   **`onBrokenMarkdownLinks` (string)**: Action to take when a broken Markdown link is found.
-   **`favicon` (string)**: Path to the website favicon.
-   **`organizationName` (string)**: GitHub organization name (if applicable).
-   **`projectName` (string)**: GitHub project name (if applicable).
-   **`i18n` (object)**: Internationalization configuration.
    -   `defaultLocale` (string)
    -   `locales` (array of strings)
-   **`presets` (array of objects)**: Defines the Docusaurus presets used.
    -   `'@docusaurus/preset-classic'` (string)
    -   **`docs` (object within `preset-classic`)**: Configuration for the documentation plugin.
        -   `sidebarPath` (string): Path to the `sidebars.ts` file.
        -   `editUrl` (string): URL to edit documents on GitHub (if applicable).
        -   `showLastUpdateAuthor` (boolean)
        -   `showLastUpdateTime` (boolean)
        -   `routeBasePath` (string): The base path for docs (default `/docs`).
    -   **`blog` (object within `preset-classic`)**: Configuration for the blog plugin (can be removed if not used).
    -   **`theme` (object within `preset-classic`)**: Configuration for the theme plugin.
-   **`themeConfig` (object)**: Main theme configuration.
    -   **`navbar` (object)**: Navigation bar configuration.
        -   `title` (string)
        -   `logo` (object): Logo details.
            -   `alt` (string)
            -   `src` (string)
        -   `items` (array of objects): Navigation links.
            -   `type` (string, e.g., `'docSidebar'`)
            -   `sidebarId` (string, for `docSidebar` type)
            -   `position` (string, e.g., `'left'`)
            -   `label` (string)
            -   `href` (string)
    -   **`footer` (object)**: Footer configuration.
    -   **`prism` (object)**: Code block syntax highlighting.
    -   **`colorMode` (object)**: Dark/light mode configuration.
-   **`stylesheets` (array of objects)**: Custom stylesheets.

### Validation Rules:
-   `sidebarPath` MUST point to a valid `sidebars.ts` file.
-   `baseUrl` MUST end with a trailing slash.
-   `onBrokenLinks` and `onBrokenMarkdownLinks` SHOULD be set to `'throw'` during development for strict link checking.
-   `navbar.items` SHOULD include a link to the main documentation (textbook).

## 2. `sidebars.ts`

This TypeScript file defines the structure of the documentation sidebar. It typically exports a default object containing one or more sidebar configurations.

### Key Sections and Their Structure:

-   **`tutorialSidebar` (array of objects)**: An example sidebar ID. In our case, this will be structured to represent modules and chapters.
    -   Each object in the array can be:
        -   **Category**: Represents a module.
            -   `type` (string): `'category'`
            -   `label` (string): Display name of the module (e.g., "Module 1: ROS 2").
            -   `items` (array of objects/strings): Chapters within the module.
                -   Each item can be a string (referencing a Markdown file ID) or another category for sub-sections.
                -   `type` (string): `'doc'`
                -   `id` (string): Document ID (derived from filename without extension).
        -   **Document**: Represents a single chapter.
            -   `type` (string): `'doc'`
            -   `id` (string): Document ID.
        -   **Autogenerated**: For automatically generated sidebars.
            -   `type` (string): `'autogenerated'`
            -   `dirName` (string): The directory to auto-generate from.

### Validation Rules:
-   The sidebar MUST define 4 top-level categories representing modules.
-   These categories MUST contain a total of 14 document entries (chapters).
-   Each document `id` MUST correspond to an existing Markdown file within the `docs` directory structure.
-   Labels SHOULD be clear and descriptive for navigation.

## 3. `docs/` Directory Structure

This is not a configuration file itself, but its structure directly influences `sidebars.ts` and content organization.

### Key Structure:

```
docs/
├── Module-1-ROS 2/                   # Top-level directory for Module 1
│   ├── _category_.json               # Metadata for Module 1
│   ├── introduction-to-physical-ai.md
│   ├── physical-ai-landscape.md
│   ├── python-and-ros-2.md
│   └── ros-2-fundamentals.md
├── Module-2-Gazebo-Unity/            # Top-level directory for Module 2
│   ├── _category_.json               # Metadata for Module 2
│   ├── advanced-simulation-with-unity.md
│   ├── simulating-a-robot-senses.md
│   ├── simulating-robots-in-gazebo.md
│   └── understanding-urdf-for-humanoids.md
├── Module-3-ISAAC/                   # Top-level directory for Module 3
│   ├── _category_.json               # Metadata for Module 3
│   ├── advanced-path-planning-with-nav2.md
│   ├── introduction-to-nvidia-isaac-sim.md
│   └── visual-slam-and-navigation-with-isaac-ros.md
└── Module-4-VLA/                     # Top-level directory for Module 4
    ├── _category_.json               # Metadata for Module 4
    ├── capstone-project.md
    ├── cognitive-planning-with-llms.md
    └── voice-to-action-with-openai-whisper.md
```

### Validation Rules:
-   There MUST be 4 module directories within `docs/`.
-   Each module directory SHOULD contain a `_category_.json` file.
-   The total count of Markdown files across all module directories MUST be 14.
-   All Markdown files MUST have unique IDs (derived from their filenames within their respective module paths).
